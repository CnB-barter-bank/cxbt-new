# CXBT - ERC-20 токен с возможностью минтинга и паузы

## Описание проекта

CXBT - это стандартный токен ERC-20, разработанный на платформе Ethereum с использованием библиотек OpenZeppelin. Контракт реализует все основные функции стандарта ERC-20, а также включает дополнительные возможности:

- **Минтинг токенов**: Владелец контракта может создавать новые токены в любое время
- **Пауза**: Владелец может приостановить все переводы токенов в экстренных ситуациях
- **Минтинг при паузе**: Минтинг токенов доступен даже когда контракт на паузе
- **Управление правами**: Только владелец контракта может выполнять привилегированные операции

## Технологии

- **Solidity** ^0.8.20 - язык программирования смарт-контрактов
- **Hardhat** - фреймворк для разработки Ethereum-приложений
- **OpenZeppelin** - библиотека безопасных смарт-контрактов
- **Ethers.js** - библиотека для взаимодействия с Ethereum
- **Chai** - библиотека для тестирования

## Установка зависимостей

Для начала работы с проектом необходимо установить зависимости:

```bash
npm install
```

## Компиляция контрактов

Для компиляции смарт-контрактов выполните:

```bash
npm run compile
```

Скомпилированные файлы будут сохранены в директории `artifacts/`.

## Запуск тестов

Для запуска всех тестов выполните:

```bash
npm run test
```

Проект включает 47 тестов, покрывающих все основные функции контракта:

- Деплой контракта
- Перевод токенов (transfer)
- Разрешение и перевод от имени (approve, transferFrom)
- Минтинг токенов (mint)
- Пауза и разпауза (pause, unpause)
- Блокировка переводов при паузе
- Проверка прав доступа
- Интеграционные тесты

## Деплой контракта на локальную сеть

### Запуск локальной сети

Для запуска локальной сети Hardhat в отдельном терминале:

```bash
npx hardhat node
```

### Деплой контракта

В новом терминале выполните:

```bash
npx hardhat run scripts/deploy.js --network localhost
```

После успешного деплоя вы увидите адрес контракта:

```
CXBT deployed to: 0x5FbDB2315678afecb367f032d93F642f64180aa3
```

## Функции контракта

### Основные функции ERC-20

- `name()` - Возвращает название токена
- `symbol()` - Возвращает символ токена (тикер)
- `decimals()` - Возвращает количество десятичных знаков (18)
- `totalSupply()` - Возвращает общее количество токенов
- `balanceOf(address)` - Возвращает баланс указанного адреса
- `transfer(address, uint256)` - Переводит токены на другой адрес
- `allowance(address, address)` - Возвращает разрешённое количество токенов
- `approve(address, uint256)` - Разрешает другому адресу тратить токены
- `transferFrom(address, address, uint256)` - Переводит токены от имени другого адреса

### Дополнительные функции

- `mint(address to, uint256 amount)` - Минтит новые токены на указанный адрес (только владелец)
- `pause()` - Приостанавливает все переводы токенов (только владелец)
- `unpause()` - Возобновляет переводы токенов (только владелец)

### События

- `Transfer(address indexed from, address indexed to, uint256 value)` - Событие перевода токенов
- `Approval(address indexed owner, address indexed spender, uint256 value)` - Событие разрешения
- `Paused(address account)` - Событие приостановки контракта
- `Unpaused(address account)` - Событие возобновления работы контракта

## Конфигурация

Параметры токена задаются при деплое:

- `name` - Название токена (например, "My Token")
- `symbol` - Символ токена (например, "MTK")
- `initialSupply` - Начальное количество токенов (например, 1,000,000 токенов)

## Безопасность

- Контракт использует проверенные библиотеки OpenZeppelin
- Минтинг и управление паузой доступны только владельцу контракта
- При паузе все переводы токенов блокируются
- Минтинг доступен даже при паузе для гибкости управления
